# 单层时间轮定时器（Single-Level Time Wheel）

## 一、简介

时间轮（Time Wheel）是一种高效的定时器管理数据结构，最早由 Varghese 和 Lauck 在论文 **Hashed and Hierarchical Timing Wheels** 中提出，目前广泛应用于网络服务器、RPC 框架和消息队列等场景。

与升序链表、最小堆等维护全局有序性的方案不同，时间轮将时间划分为固定大小的时间片（Tick），使用一个**环形数组（Circular Array）**模拟时间流逝，每个数组元素称为一个**槽（Slot）**。

时间轮中的定时器不会按照超时时间排序，而是根据超时时间计算其所在的槽（Slot）和剩余需要等待的圈数（Round）。当时间轮转动到对应槽时，再根据 Round 判断定时器是否真正到期。

因此，时间轮能够以 **O(1)** 的时间复杂度完成定时器的插入，非常适合管理大量定时任务。

# 二、时间轮结构

假设时间轮共有 **60 个槽**：

```text
                [0]
           /             \
       [59]               [1]
      /                     \
    [58]                     [2]
     |                        |
    ...                      ...
      \                     /
       [31] ... [30] ... [3]
```

每个槽表示固定的时间单位，例如：

- 一个槽 = 1 秒（1 Tick）
- 整个时间轮 = 60 秒

时间轮维护一个当前槽指针：

```cpp
current = 0;
```

每调用一次 `tick()`：

```cpp
current = (current + 1) % slotCount;
```

表示时间向前推进 **一个 Tick**，随后处理当前槽中的所有定时器。

> **说明：**
>
> 本实现约定：
>
> `timeout >= 1`
>
> 表示**经过 `timeout` 个 Tick 后执行定时器**。

# 三、定时器如何存放

例如：

当前：

```text
current = 10
```

添加一个：

```text
15 Tick 后执行
```

计算目标槽：

```cpp
index = (current + timeout) % slotCount
      = (10 + 15) % 60
      = 25
```

因此，将该定时器放入：

```text
slot[25]
```

中。

整个过程无需遍历，也无需排序，因此插入复杂度为 **O(1)**。

# 四、为什么需要 Round

如果：

```text
timeout = 80 Tick
```

而时间轮只有：

```text
60 个槽
```

则：

```cpp
index = (10 + 80) % 60
      = 30
```

如果仅记录槽的位置，那么时间轮第一次转到：

```text
slot[30]
```

时就会错误执行。

因此，需要额外记录：

```cpp
round = (timeout - 1) / slotCount
      = (80 - 1) / 60
      = 1
```

表示：

> **第一次经过该槽时不能执行，还需要等待一整圈；第二次经过该槽时才真正执行。**

Timer 定义如下：

```cpp
struct Timer
{
    uint32_t round = 0;

    std::function<void()> callback;
};
```

其中：

- `round`：剩余需要等待的完整圈数。
- `callback`：定时器超时后的回调函数。

# 五、Tick 工作流程

每调用一次：

```cpp
tick();
```

时间轮完成以下两个步骤：

```text
① current 向前移动一个槽

↓

② 检查当前槽中的所有定时器
```

对于每个 Timer：

```text
round > 0
    ↓
round--

round == 0
    ↓
执行 callback
    ↓
删除 Timer
```

例如：

```text
slot[20]

↓

TimerA  round = 0

TimerB  round = 2

TimerC  round = 1
```

当时间轮转动到该槽时：

```text
TimerA
↓

执行并删除

TimerB
↓

round = 1

TimerC
↓

round = 0
```

下一次时间轮再次转动到：

```text
slot[20]
```

时：

```text
TimerC
↓

执行
```

再下一圈：

```text
slot[20]
```

时：

```text
TimerB
↓

执行
```

整个过程中，仅处理当前槽中的定时器，而不会遍历整个时间轮。

# 六、为什么时间轮能够做到 O(1)

时间轮最大的优势就是**插入效率高**。

对于升序链表、最小堆等结构，插入一个新的定时器时，都需要维护全局有序性，因此需要寻找插入位置：

- 升序链表需要遍历链表寻找位置。
- 最小堆需要进行上浮操作维持堆序。

而时间轮完全不同。

当添加一个新的定时器时，只需要根据超时时间计算：

```cpp
index = (current + timeout) % slotCount;

round = (timeout - 1) / slotCount;
```

随后直接插入对应槽即可。

整个过程：

- 不需要比较已有定时器。
- 不需要维护全局排序。
- 不需要移动其它节点。

因此，时间轮插入复杂度始终为：

```text
O(1)
```

这也是时间轮能够管理大量定时任务的根本原因。

# 七、复杂度分析

| 操作 | 时间复杂度 |
|------|-----------:|
| 添加定时器 | O(1) |
| 删除定时器 | O(1) |
| Tick | O(k) |

其中：

```text
k = 当前槽中的定时器数量
```

由于无需维护全局有序性，因此插入定时器仅需计算一次槽的位置即可完成。

# 八、优缺点

## 优点

- 添加定时器复杂度为 **O(1)**。
- 删除定时器复杂度为 **O(1)**。
- 无需维护全局排序。
- 非常适合管理大量定时任务。
- 数据按槽组织，局部性较好，实现简单。

## 缺点

- 时间精度受 Tick 大小限制。
- Tick 越小，时间轮越大，占用内存越多。
- Tick 越大，定时精度越低。
- 单层时间轮适合处理中短时间定时任务，不适合跨度特别大的超时。

因此，工业实现通常采用：

- 多层时间轮（Hierarchical Time Wheel）
- 哈希时间轮（Hashed Timing Wheel）

来支持更大的时间跨度。

# 九、与升序链表比较

| 项目 | 升序链表 | 单层时间轮 |
|------|---------:|-----------:|
| 插入 | O(n) | O(1) |
| 删除 | O(1) | O(1) |
| Tick | O(1)（检查链表头） | O(k) |
| 是否维护全局排序 | 是 | 否 |
| 是否需要寻找插入位置 | 是 | 否 |

其中：

- **升序链表**适合定时器数量较少、超时时间变化频繁的场景。
- **单层时间轮**适合网络服务器、连接管理等拥有大量定时任务的场景。
